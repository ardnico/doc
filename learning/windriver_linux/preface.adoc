== 序章: 学習の目的と全体像
Wind River Linux(WRL)はWind River Systemsが商用サポートと長期提供(LTS)を組み合わせて提供する組み込みLinuxディストリビューションです。Yocto Projectを基盤とすることで柔軟性を維持しつつ、産業用途で要求される認証、長期保守、セキュリティパッチ提供を実現しています。本序章では、学習の狙いと以降の章で扱うトピックの位置付けを明確にします。

=== 想定読者
* Yocto ProjectまたはBitBakeによる組み込みLinuxビルド経験を持ち、Wind River Linux特有の拡張を把握したいエンジニア。
* 産業機器、通信、車載分野などで長期保守可能なLinuxプラットフォームを評価・導入する技術担当者。
* 社内BSP開発・運用チームとの連携や、セキュリティ部門との調整を担うプロジェクトリーダー。

=== 学習目標
本ドキュメントを読み進めることで、以下を達成することを目指します。

* Wind River Linuxが提供する価値(サポート、ツール、検証済みレイヤ)を理解し、導入判断に必要な視点を得る。
* WRLのビルドプロジェクト構成と標準Yocto Projectとの差異を把握し、開発プロセスを設計できる。
* BSP開発、セキュリティ対策、規格準拠、運用保守までのライフサイクル全体を俯瞰し、チーム間連携に必要な共通語彙を揃える。

=== Wind River Linuxの価値
[%autowidth.stretch,options="header"]
|===
|視点|提供価値|補足
|開発効率|`wrlinux-setup`によるプロジェクト生成、Wind River検証済みレイヤ|Yocto互換のため既存レシピ資産が活用可能
|品質保証|長期サポート、定期的なCVEパッチ、テスト済み BSP|商用サポート契約により迅速な不具合エスカレーションが可能
|運用継続性|ライフサイクル管理ツール、Field Support Services|運用フェーズでのホットフィックスや現地調査支援が受けられる
|===

[NOTE]
====
*Yocto Projectとの関係*: WRLはYocto Project Compatible製品として認証されており、`poky`や`meta-openembedded`などの既存レイヤとの互換性が保たれています。
====

=== 学習の進め方
本ガイドでは章ごとにテーマを分割し、以下の観点で学習できるよう構成しています。

1. 環境構築: 推奨ホスト要件、ライセンス取得、ツールチェーン準備。
2. ビルドシステム: `wrlinux-setup`、BitBake、`conf/`構成の理解。
3. レイヤカスタマイズ: Wind River提供レイヤの整理と社内レイヤ設計。
4. デバッグとプロファイリング: 解析ツールとログ活用。
5. BSP開発: ハードウェアサポート、デバイスツリー、ドライバ統合。
6. セキュリティ: 脆弱性対応、SBOM生成、ランタイム防御。
7. 規格準拠: IEC 62304、ISO 26262などの準拠ポイント。
8. 運用とライフサイクル: アップデート戦略、フィールドサポート。

各章では専門用語に対する補足、関連ドキュメントへのリンク、実践的なチェックリストを盛り込み、学習後にすぐ実務へ適用できる内容を目指します。
